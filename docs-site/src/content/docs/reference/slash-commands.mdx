Los slash commands se escriben en el input del chat con `/`. Son **exclusivos del chat** — cualquier accion de configuracion, diagnostico o navegacion vive en el [command palette](/funcionalidades/command-palette/) bajo `:` (shift + :).

:::note[Separacion estricta `/` vs `:`]
- **`/`** — solo opera sobre el turno actual, historial de chat, contexto AI, agents, todos y memoria persistente.
- **`:`** — configuracion, diagnostico, navegacion, exploradores y modales (theme, model, autoskill).

Si un slash no aparece aqui, probablemente migro a `:`. El chat responde con hint de migracion.
:::

## Sesion (10)

| Comando | Descripcion |
|---------|-------------|
| `/clear` | Limpia el historial del chat y reinicia la conversacion desde cero |
| `/exit` | Cierra el chat y vuelve a la pantalla inicial de ingenierIA |
| `/resume` | Retoma la ultima sesion de chat exactamente donde la dejaste |
| `/history` | Muestra las sesiones guardadas para retomar |
| `/fork <label>` | Ramifica la sesion actual con un label para explorar otro camino |
| `/export [path]` | Exporta la sesion actual como archivo JSONL compartible |
| `/compact [strategy]` | Compacta mensajes anteriores para liberar contexto. Estrategias: `aggressive` (a), `balanced` (b, default), `conservative` (c) |
| `/undo` | Deshace el ultimo turn (pop user + assistant, restaura draft) |
| `/redo` | Rehace el ultimo `/undo` (solo si no enviaste un nuevo mensaje) |
| `/continue` | Reintenta el ultimo turno del AI sin enviar un nuevo prompt |

## Migracion desde slashes antiguos

Los siguientes slashes fueron **eliminados** o **movidos a `:` palette**. Si escribes uno, el chat responde con hint de redireccion:

| Slash antiguo | Nuevo acceso |
|---------------|--------------|
| `/theme` | `:theme` → abre ThemePicker (6 variantes) |
| `/model` / `/change-model` | `:model` → abre ModelPicker |
| `/autoskill` | `:autoskill` → abre modal con checkboxes |
| `/install-skills` | absorbido por el modal de `:autoskill` |
| `/doctor` | `:doctor` (absorbe lsp-status, lsp-diag, bridge-status, features, mcp-detail) |
| `/mcp-status` | `:mcp-status` |
| `/audit` | `:audit` |
| `/skills` / `/commands` / `/adrs` / `/policies` / `/agents` / `/workflows` | `:<id>` en exploradores |
| `/permissions` | `:permissions` (cicla Standard → Permissive → Strict) |
| `/hooks`, `/features`, `/lsp-*`, `/bridge-status` | absorbidos por `:doctor` |
| `/compliance`, `/install-skills`, `/go`, `/fork-from`, `/load`, `/memories`, `/brief`, `/sessions`, `/retry`, `/todo-check` | eliminados (redundantes o no usados) |

## Autocomplete

Al escribir `/` en el input del chat, aparece un popup con los 42 comandos que coinciden con lo que estas tecleando. Cada entrada muestra nombre y descripcion. Navega con `↑`/`↓` y confirma con `Enter` o `Tab`.

## Select-then-type (workflows)

Al seleccionar un workflow:

1. El nombre se inserta como prefijo en el input (ej: `/add-feature `)
2. El prefijo se muestra en **cyan bold**
3. Escribes tu peticion y `Enter` la envia con el workflow cargado

```
/add-feature crear endpoint de pagos con validacion
```

Si presionas Enter sin argumento, el workflow queda cargado esperando tu input. Al borrar el prefijo, el workflow se limpia automaticamente.
